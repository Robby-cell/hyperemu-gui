# HyperEmu GUI

A cross-platform, interactive graphical emulator and assembly debugger built in Rust using [`egui`](https://github.com/emilk/egui) and the [`hyperemu`](https://github.com/Robby-cell/hyperemu) engine. 

HyperEmu allows you to write assembly code, compile it on-the-fly, and step through execution while visually inspecting CPU registers, memory, and memory-mapped hardware peripherals. It natively supports Desktop (Windows, macOS, Linux) and Web (WebAssembly).

## ✨ Features

- **Multi-Architecture Support:** Seamlessly switch between different 32-bit architectures.
  - **ARMv7 (32-bit)**
  - **RISC-V (RV32I)**
  - **x86 (32-bit i386)**
- **Integrated IDE & Debugger:**
  - Built-in assembly code editor with syntax highlighting, line numbers, and error reporting.
  - Live Disassembly view with byte inspection and internal AST (Abstract Syntax Tree) decoding.
  - Interactive debugging: set breakpoints, pause, run, and single-step through instructions.
- **Hardware & Peripheral Emulation:**
  - **Dynamic Memory Map:** Add, configure, and remove RAM, GPIO, and UART peripherals dynamically via the UI.
  - **Interactive GPIO:** Click virtual buttons and watch virtual LEDs toggle in real-time as your assembly interacts with hardware registers.
  - **UART Console:** Live text console for `stdout`-like printing from your assembly programs.
- **Deep System Inspection:**
  - **Live CPU Registers:** Watch registers update in real-time (highlighted in yellow when mutated), including architecture-specific status flags (CPSR for ARM, EFLAGS for x86).
  - **Live Stack View:** Real-time tracking of the top 16 words on the stack, automatically tracking the Stack Pointer (SP).
  - **Hex Memory Viewer:** Inspect raw memory and ASCII representations at any address.
- **Workspace Management:** Save and load your code, active memory maps, and clock speeds as JSON workspaces.
- **Cross-Platform & Responsive:** Desktop native execution and full WASM web support. Automatically adapts to mobile screens using a touch-friendly tabbed layout.

## 🚀 Getting Started

### Prerequisites
Make sure you have [Rust](https://rustup.rs/) installed.

### Running on Desktop (Native)
To run the emulator natively on Windows, macOS, or Linux:

```bash
cargo run --release
```

### Running on the Web (WebAssembly)
To compile and run the project in your browser, you'll need a tool like [`trunk`](https://trunkrs.dev/):

```bash
# Install the web target
rustup target add wasm32-unknown-unknown

# Install Trunk
cargo install trunk

# Build and serve locally
trunk serve
```
Then, open your browser to `http://127.0.0.1:8080`.

## 🎮 Usage Guide

1. **Select Architecture:** Use the dropdown in the top navigation bar to select your target architecture (ARMv7, RISC-V, or x86). Default starter code will automatically load to demonstrate interacting with the UART and GPIO.
2. **Configure Memory:** Open the **Memory Map** tab to map your peripherals. By default, RAM (Code/Stack), a UART console (0x10000000), and a GPIO device (0x40000000) are mapped.
3. **Compile & Load:** Click **"⚙ Compile & Load"** in the top bar. This will assemble your code, map the memory, and load the binary into the virtual CPU.
4. **Execution Controls:** 
   - Use **▶ Run** to start execution.
   - Adjust the **Speed** slider dynamically from 1 Hz up to 16 MHz.
   - Use **⏸ Pause** and **⏭ Step** to debug your logic instruction by instruction.
   - Click in the editor gutter to toggle **Breakpoints** (indicated by red dots).
5. **Interact:** Open the **Hardware** and **Consoles** tabs to push buttons, view LEDs, and read serial output generated by your code.

## 📁 Project Structure

- `src/main.rs` - Application entry point, handling both Native and WASM initialization.
- `src/app.rs` - Main application state (`EmuApp`), workspace management, save/load logic, and execution loop timing.
- `src/backend/` - Architecture-specific emulation backends.
  - `armv7.rs` - ARMv7 backend STM32-style GPIO wrapper.
  - `rv32i.rs` - RISC-V backend GPIO wrapper.
  - `x86.rs` - x86 (i386) backend custom x86 GPIO device implementation.
- `src/ui/` - `egui` frontend implementation.
  - `mod.rs` - Main layout logic, responsive design (Desktop vs. Mobile).
  - `panels.rs` - Implementation of the specific views (Editor, Disassembly, Registers, Stack, Memory Map).
  - `peripherals.rs` - GUI wrappers for hardware rendering (e.g., UART terminal).

## 🛠 Dependencies
- `eframe` / `egui` - Immediate mode GUI.
- `hyperemu` - The core emulation engine, memory bus, and generic device traits.
- `armv7_encoder` / `armv7_disassembler` - ARM parsing.
- `riscv_asm` - RISC-V assembly handling.
- `x86_translator` - x86 assembly and disassembly.
- `serde` - Workspace state serialization.
- `rfd` - Native and WASM asynchronous file dialogs.
